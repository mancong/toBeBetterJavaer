# 选题调研 Skill

目标：每次运行产出 5 个当天可写的候选选题，每个选题都要有**热点依据**（今天为什么值得写）和**数据依据**（历史上同类文章的打开率证明读者爱看）。

## 工作流程

### 第一步：确认日期

```bash
date "+%Y年%m月%d日"
```

热点有时效性，所有搜索都以这个日期为基准。

### 第二步：分析历史数据

```bash
python3 .claude/skills/topic-research/scripts/analyze_history.py
```

输出按打开率倒序的历史文章清单（打开率 = 阅读人数/送达人数，分享率反映传播力）。**每次都要重新归纳品类规律**，不要依赖下面的静态结论——xlsx 会持续更新，以最新脚本输出为准。

归纳时回答三个问题：

- 打开率前 10 的文章覆盖哪些品类？
- 分享率高（≥2%）的文章有什么共性？（分享率高说明有传播力，容易带来新关注）
- 打开率垫底的文章踩了什么坑？（避免推荐同类选题）

作为参考基线，截至 2026 年 7 月的数据显示读者偏好从高到低为：

1. **国产模型重大发布 + 实测**（如 DeepSeek V4、Kimi 开源，打开率最高可到 50%+）
2. **热点事件技术拆解**（如截图暗水印溯源，蹭上全民话题时爆发力强）
3. **面试/职场对话体 + AI 技术八股**（Agent、RAG、MCP、Skill 概念拆解，稳定 10%~20%）
4. **Claude Code 原理与技巧**（grep vs RAG、CLAUDE.md 维护等，打开率中等但分享率高）
5. **工具教程/薅羊毛类**（打开率明显偏低，除非工具本身是大热点，否则少推荐）

### 第三步：查重

避免推荐最近已经写过的题材：

在项目根目录执行：

```bash
git log --oneline --since="14 days ago" -- docs/src
ls -t docs/src/ai/video/ | head -10
```

再看一眼 `docs/src/sidebar/itwanger/` 下最近修改的文件。同一个主题两周内写过的，除非有重大新进展，否则不再推荐。

### 第四步：当天热点调研

执行 web-access Skill 抓取热点（所有联网操作走 web-access，不用 WebFetch）。按优先级搜这些源：

| 来源 | 搜什么 |
|---|---|
| 官方发布 | Anthropic、OpenAI、DeepSeek、月之暗面、阿里通义、智谱的博客/发布页，有没有新模型、新功能 |
| 互联网大厂动态 | 阿里、字节、腾讯、华为、百度、京东、美团最近 48 小时的 AI 相关新闻：新产品、开源项目、组织架构调整、薪酬招聘动向、高管表态（薪酬/招聘类历史打开率高，单独盯） |
| 同行公众号 | 用搜狗微信搜索看同行博主最近发了什么（见下方专门说明） |
| GitHub Trending | 当天/本周冲榜的 AI 项目，重点看 Agent、CLI 工具、开发框架。**在榜 ≠ 新热点**：老项目常年挂在 Trending 上，每个候选项目必须用 `curl -s https://api.github.com/repos/<owner>/<repo>` 查 `created_at`，创建超过 2 周的项目除非有 48 小时内的重大 release（查 `/releases/latest`），否则不作热点依据 |
| Hacker News | 首页 AI 相关高热帖 |
| 国内技术媒体 | 掘金热榜、量子位、机器之心当天头条 |
| X/Twitter | AI 圈大 V 在讨论什么（karpathy、官方账号等） |

每条热点记录：事件本身、发生时间、信源链接。**只要 48 小时内的热点**，超过 3 天的不算热点。

#### 同行公众号调研（搜狗微信搜索）

微信生态的内容谷歌/百度搜不到，搜狗微信搜索是少数入口，但它有反爬限制，必须走 browser-act 的 stealth 浏览器（已验证可用）：

```bash
# 找到 stealth 类型的浏览器 ID（机器上已有现成的）
browser-act browser list

# 搜"关键词 + 博主名"，type=2 表示搜文章
browser-act --session topic-research browser open <stealth浏览器ID> \
  "https://weixin.sogou.com/weixin?type=2&query=AI+Agent+苍何"
browser-act --session topic-research wait stable
browser-act --session topic-research get markdown
```

返回结果含标题、摘要、公众号名、发布日期和翻页链接。使用要点：

- 同行博主清单（逐个用博主名作关键词搜，可根据用户指示增删）：苍何、JavaGuide、stormzhang、赛博禅心、小林coding、cxuanAI、TRAE.ai、GitHubDaily、小金AI、代码随想录、打工没有自由、宫水三叶的刷题日记、刘小排r、AI普瑞斯
- **只看最近两天发布的文章**，更早的直接丢弃，不进调研结果
- 搜博主名会混进"提到该博主"的文章，以结果里的公众号名为准做过滤
- 博主名之外，也可以直接搜热点词（如"DeepSeek"、"Claude Code"）看全网公众号最近谁在写、怎么写
- 不要在 URL 里直接拼 `tsn` 时间参数（会被重定向回首页），时间过滤靠结果里自带的发布日期做；需要翻页时用结果页底部的"下一页"链接
- 结果按相关度排序而非时间，第一页没有两天内的就翻到第二页确认，再没有就换下一个博主
- 清单有十多个博主，全部搜完再汇总，中途不要因为某个博主没结果就停
- 调研结束后 `browser-act session close topic-research` 关闭会话

### 第五步：打分与筛选

对每条热点做三维评估：

- **热度**：是不是圈内都在讨论？有没有争议点或反常识点？
- **匹配度**：落在第二步归纳出的高打开率品类里吗？
- **可写性**：今天一天内能不能完成调研+实测+成稿？需要实测的工具能不能装上？

三项都过关的进入候选。候选不足 5 个时，可以从"历史高打开率品类 + 无时效热点"里补位（比如 Agent 八股拆解类，任何时候都能写）。

不进候选的类型（用户已明确反馈）：

- 人物任命/组织架构调整类（如"某大厂 Agent 业务换帅"）——和读者的技术/求职诉求关联太弱，历史数据里也没有此品类的高打开率模板
- 营收/估值/商业格局类（如"A 公司营收反超 B 公司"）——读者不关心厂商的生意，除非能落到工具限额、价格等直接影响使用的变化
- 创建超过 2 周且无重大新 release 的 GitHub 项目（见上表 Trending 一行）

### 第六步：输出选题报告

按推荐优先级输出 5 个候选，每个候选包含：

```markdown
## 候选 N：选题方向一句话

- **热点依据**：什么事件 + 信源链接 + 发生时间
- **标题模板**：引用的历史标题原文（一字不差）+ 它的打开率 xx%
- **建议标题**：按模板完形填空得出的新标题（只换专有名词和数字，其余逐字保留，字数差 ≤2）
- **推荐风格**：ai-article 四种风格选一（安装教程/产品评测/面试对话/深度拆解）
- **切入角度**：具体写什么，和别人可能的写法差异在哪
- **素材清单**：写作前需要抓取/实测的东西
```

标题模板的选取原则：优先从打开率前 15 的标题里挑品类匹配的；建议标题必须给出，让用户看一眼就能判断这个选题"长成标题什么样"。

完形填空的硬规则（用户已明确反馈，违反任何一条都要换模板重来）：

- **先识别模板的语义槽位再替换**。比如《面试官："没用过 Claude Code 也敢来？"……最新模型 Mythos 你可懂？》里，Claude Code 是文章主体工具（内容围绕它写），Mythos 是热点引子——替换时槽位必须对应，不能把主体槽位的词换成热点词
- **"X 出手了"类模板的主语必须是互联网公司/厂商**（腾讯、字节、阿里……），不能换成网信办、商务部等政府机构——读者身份是互联网技术人，政府主语与读者无关
- **同一轮报告里每个候选用不同的模板**，不得复用
- **替换后的标题必须语义自然、概念真实存在**，不硬造说法（反例："DeepSeek V4 的迁移"——迁移的是接口不是模型，语义不通就不要用）
- 拿不准时宁可标注"此候选无合适模板，标题写作时再定"，不硬套

报告末尾问用户选哪个。用户选定后，把该候选的素材清单和切入角度作为输入，执行 ai-article Skill 开写。

## 注意事项

- 数据文件复用 `../ai-article/references/gongzhonghao.xlsx`，用户更新 xlsx 后本 Skill 自动生效，不要另存副本
- 选题方向的表述遵循写作规范：书面用语、语义准确，英文术语首次出现标中文
- 不推荐需要引用过时模型数据（GPT-3.5 时代）的选题
