这套隐写术走两条路径。

第一条，读操作系统时区。代码里会识别两个值：Asia/Shanghai 和 Asia/Urumqi。命中任何一个，就会标记为中国用户。

第二条，读 ANTHROPIC_BASE_URL 环境变量。用中转站的用户都得改这个地址。Claude Code 拿到地址后，提取域名，跟一份 147 条的内置黑名单做比对。

这份黑名单不是明文存储的。它用 Base64 加上 XOR 异或运算做了混淆，密钥是 91。解码之后，里面全是国内公司的域名——百度、字节、阿里、京东、B 站、小红书，还有 DeepSeek、月之暗面、MiniMax 这些 AI 公司。甚至连几十个中转站的域名都在里面。

好，重点来了。检测完之后怎么标记？

不加字段，不发额外请求，而是改系统提示词里一句普通的日期——"Today's date is 2026-06-30"。

改两个地方。

第一个，Today's 里的撇号。正常是 ASCII 单引号，检测到中国域名就换成右单引号 U+2019，检测到 AI 实验室关键词就换成修饰字母撇号 U+02BC。四种字符，肉眼完全看不出区别，但 Anthropic 后端一查 Unicode 编码就知道了。

第二个，日期分隔符。命中时区就把连接字符换成斜杠，2026-06-30 变成 2026/06/30。

两处合在一起，3 个比特，8 种组合。精确到能分辨"用百度内网的员工"和"用 DeepSeek 中转的独立开发者"。

为什么比查 IP 狠？因为标记藏在正常请求里，不是额外的网络请求，中转站解决不了，抓包也看不出异常。这就是隐写术和密码学的区别——密码学藏内容，隐写术藏存在本身。

怎么绕过这套机制呢？改时区，改成纽约或伦敦，换一个不在黑名单里的中转站域名，或者用本地代理拦截请求把日期格式还原。

最后一句口诀——时区域名双路检，撇号斜杠藏玄机；改时区换域名，三个比特全归零。
