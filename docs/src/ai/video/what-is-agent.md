拿到这道题，先想清楚面试官考什么。

他问"什么是 Agent"，表面在问概念，实际考三件事：第一，你对 Agent 的理解是不是架构级的；第二，你能不能说清它和裸调 API 的本质区别；第三，你有没有真正搭过 Agent。

好，接下来给你满分回答，照着背就完事了。三个本质区别。

第一个，一问一答 vs 循环决策。

裸调 API 是什么？发一条 prompt，拿一条 completion，结束。下一步做什么，你的代码说了算，流程写死在业务逻辑里。

Agent 不一样。核心是一个 Think-Act-Observe 循环——LLM 自己想下一步做什么，执行一个动作，拿到环境反馈，再想，再做，直到任务完成。控制权在 LLM 手里，不在你的代码里。这是最本质的区别。

第二个，只会说 vs 能动手。

裸调 API，用户问"北京天气怎么样"，模型只能从训练数据里编一个答案，可能是去年的，也可能纯属虚构。

Agent 呢？LLM 判断"我需要实时数据"，通过 Function Calling 返回 tool_calls，指定调用 query-weather，参数 city 等于北京。Agent 执行调用，拿到真实结果——晴，32℃，再基于真实数据生成回答。

同样的问题，裸调靠猜，Agent 靠查。

第三个，失忆 vs 有状态。

裸调 API 无状态，每次调用相互独立，上一轮聊了什么它不知道，除非你手动把对话历史拼进 prompt。

Agent 在循环中自动维护上下文——调过什么工具、拿到什么结果、还差什么没完成，全部记着。这不是你手动管理的，是循环机制自带的。

面试官如果追问："基础 RAG 算不算 Agent？"

不算。基础 RAG 是固定管线：检索文档、塞进 prompt、生成回答，每一步都是代码写死的。什么时候才算 Agent？当 LLM 自己决定要不要检索、用什么 query、结果够不够用、要不要换个关键词再搜一轮——有了这个决策循环，才叫 Agent。

最后一句口诀——裸调一问一答会失忆，Agent 循环调用能动手。
